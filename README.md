# Text-Based Asset Pricing: WSJ Headlines, LLM Topics, and Embeddings

## Project Overview

This project investigates whether textual information from **Wall Street Journal (WSJ) headlines** can explain stock market returns and forecast macroeconomic activity. Using a dataset of ~10,200 headlines spanning January 1984 to December 2017, we compare three progressively richer text representation approaches:

1. **Sparse Word Counts** — A document-term matrix of 18,093 unique terms aggregated to monthly frequency
2. **LLM-Generated Topics** — 1–3 keyword topics per headline generated by **Google Gemini 2.5 Flash** (2,025 unique terms after aggregation)
3. **Dense Text Embeddings** — 250-dimensional vectors from **Google Gemini Embedding 001**, averaged to monthly means

Each representation is evaluated on two tasks:
- **In-sample contemporaneous analysis**: Explaining monthly market returns (`mret`) using Lasso variable selection (k=5) followed by OLS regression
- **Out-of-sample forecasting**: Predicting 1-month-ahead industrial production growth (`indprol1`) using expanding-window Ridge regression

## Summary of Key Findings

### In-Sample Analysis — Market Returns (`mret`)

| Representation | Features | R² (k=5) |
|----------------|----------|----------|
| Pre-built Topics | 180 | 0.1079 |
| Raw Word Counts | 18,093 | 0.1971 |
| LLM-Generated Topics | 2,025 | 0.1302 |
| **Embeddings** | **250** | **0.1055** |

**Key Insight**: At the standard k=5 constraint, embeddings appear comparable to pre-built topics. However, dense embeddings distribute semantic information across all 250 continuous dimensions — artificially limiting them to just 5 non-zero coefficients severely underutilizes their capacity. When allowed more dimensions, embeddings dramatically outperform every other representation:

| k (Lasso) | Embedding R² | Comparison |
|-----------|-------------|------------|
| 5 | 0.1055 | Matches pre-built topics |
| 14 | 0.2009 | Surpasses raw counts (0.1971) |
| **19** | **0.2234** | **Best of any representation** |

This demonstrates that dense embeddings encode richer, more distributed semantic information than any sparse representation, but require sufficient dimensionality to express it.

### Prompt Engineering Experiments (Part 2b)

Three experiments evaluated how prompt design affects LLM topic quality:

- **Persona (Bull vs. Bear)**: The neutral/base prompt (R² = 0.1302) outperformed both the Bull persona (R² = 0.0773) and Bear persona (R² = 0.1218). Persona framing introduced systematic bias — the Bull prompt produced 21% empty outputs, and both personas selected less informative topic terms. **Neutral prompts are preferable for analytical tasks.**
- **Temperature (0.1 vs. 0.9)**: Low temperature yielded 20% identical outputs across 3 runs; high temperature yielded only 5%. Even at temperature 0.1, LLM outputs are **not fully deterministic**. The short 1–3 keyword format naturally constrains output diversity, moderating the temperature effect.
- **Lookahead Bias**: 7/10 GFC-era articles (2007–2008) produced different outputs between the basic and constrained prompts, but neither exhibited significant lookahead bias at the keyword level. The short-keyword format inherently limits the opportunity to leak future knowledge, unlike longer-form analysis tasks.

### Topic Recovery (Parts 3b & 3c)

Can the 250-dimensional "black box" embedding space reconstruct interpretable topic signals?

| Topic Source | Mean R² | Recovered (R² > 0.5) |
|-------------|---------|----------------------|
| Pre-built topics (180) | 0.6176 | **131/180 (73%)** |
| LLM-generated topics (top 50) | 0.3601 | 12/50 (24%) |

**Event-driven topics are recovered especially well**, confirming that embeddings encode newsworthy economic events:
- Financial Crisis: R² = 0.83
- Terrorism: R² = 0.76
- Recession: R² = 0.74
- Treasury Bonds: R² = 0.89

Pre-built topics are better recovered because they are carefully constructed, broad attention measures with smooth time series. LLM topics are noisier keyword counts with more sparsity and measurement error.

### Out-of-Sample Forecasting — Industrial Production Growth (`indprol1`)

| Model | Features | OOS R² |
|-------|----------|--------|
| **Embeddings (Ridge + GCV)** | **250 dims** | **-0.0469** |
| LLM Topics (Ridge + GCV) | 2,025 terms | -0.0478 |
| Historical Mean Baseline | — | -0.0565 |

All models produce **negative OOS R²**, illustrating the classic signal-to-noise challenge in macroeconomic forecasting. Text features capture **contemporaneous** events effectively (in-sample R² of 0.10–0.22 for `mret`) but struggle as **leading indicators** for real economic activity. The text tells you what happened this month, not what will happen next month.

Embeddings (-0.0469) slightly outperform LLM topics (-0.0478), and both beat the naive historical mean (-0.0565), suggesting a marginal but not economically significant predictive signal exists in the text data. The high Ridge alpha (~2,683) confirms that extreme regularization is needed, meaning most text features are noise for this forecasting task.

### Reproducing Results

> **Important**: All time-consuming API calls have already been completed. The generated data is saved locally in the provided `.parquet` files. **No new API calls are needed to reproduce the analysis.**
Note on API Keys: > API keys have been removed from the source code for security purposes. If you wish to re-run the generation or embedding scripts, please create a .env file in the root directory and add your own key as GEMINI_API_KEY="your_key_here". (However, as noted above, you can simply run the notebook using the provided .parquet files without needing an API key).


The following files contain pre-generated results:
- `articles_with_topics.parquet` — LLM-generated topics for all 10,200 headlines (Gemini 2.5 Flash)
- `articles_with_topics_bear.parquet` — Bear persona topics
- `articles_with_topics_bull.parquet` — Bull persona topics
- `articles_with_embeddings.parquet` — 250-dim embeddings for all 10,200 headlines (Gemini Embedding 001)

**To reproduce all results:**

1. Open `hw3_lasso.ipynb` in Jupyter Notebook or VS Code
2. Click **"Run All"** (or Shift+Enter through each cell)
3. All 48 cells will execute using the local `.parquet` and `.csv` files — no API keys or internet connection required

The notebook runs in approximately **2–3 minutes** on a standard machine, as it only performs local computation (Lasso paths, Ridge regressions, OLS fits) on pre-generated feature matrices.
